# 專案管理方法 (Project Management Methodologies)

專案管理方法是指用於規劃、執行和控制專案的系統化方法。不同的方法適用於不同類型的專案，現代組織經常使用混合方法。

## 📖 原文內容

### 主要的專案管理方法

#### 1. 預測性方法（Predictive / Waterfall）

**特點：**
- 前期詳細規劃
- 完整定義需求
- 順序執行
- 變更通過正式流程

**優點：**
- ✓ 易於追蹤進度
- ✓ 成本預測準確
- ✓ 文件完善
- ✓ 適合固定範圍的項目

**缺點：**
- ✗ 不靈活，難以應對變更
- ✗ 風險集中於後期
- ✗ 客戶反饋遲
- ✗ 長期項目風險高

**適用場景：**
- 建築、基礎設施
- 監管嚴格的行業
- 需求清晰且穩定的項目

#### 2. 敏捷方法（Agile）

**特點：**
- 迭代開發
- 短週期交付
- 持續反饋
- 擁抱變更

**優點：**
- ✓ 靈活應對變更
- ✓ 快速交付價值
- ✓ 持續客戶參與
- ✓ 及早發現風險

**缺點：**
- ✗ 難以預測成本
- ✗ 需要高度客戶參與
- ✗ 對團隊要求高
- ✗ 文件可能不足

**適用場景：**
- 軟體開發
- 創意項目
- 需求變化大的項目

#### 3. 混合方法（Hybrid）

**特點：**
- 結合預測和敏捷的優點
- 一些活動用預測，一些用敏捷
- 靈活調整

**例子：**
- 計劃階段使用預測
- 開發階段使用敏捷
- 根據項目特性選擇

### 方法選擇矩陣

```
變更程度
高 |  ↑ 
   |  敏捷 | 敏捷+預測
   |      |（混合）
低|  ↓ 預測 | 預測
   └─────────────────
     低          高
     需求確定性
```

### 敏捷方法的重要概念

| 概念 | 說明 |
|------|------|
| **衝刺（Sprint）** | 1-4週的迭代週期 |
| **待辦事項（Backlog）** | 待實現的需求清單 |
| **燃盡圖（Burndown）** | 顯示迴圈進度的圖表 |
| **故事點（Story Points）** | 估算工作複雜度的單位 |
| **站會（Standup）** | 每日簡短的進度會議 |

### 新增：敏捷與 PMP 的關係

```
2026 改變：
- PMP 考試接受敏捷方法
- 不是「敏捷 vs 預測」
- 而是「選擇合適的方法」
- 許多項目採用混合方法
```

---

## 🎯 重點摘要

- **沒有一種最好的方法**：選擇取決於項目特性。

- **混合方法越來越普遍**：現代組織很少使用純預測或純敏捷。

- **敏捷現在被 PMP 接受**：2026 考試涵蓋敏捷方法。

- **需求變化決定方法選擇**：高變化用敏捷，低變化用預測。

- **團隊能力很重要**：敏捷需要自組織的團隊。

## 📋 考試要點

**常見題型：**
- ✅ 選擇題：「給定情景，選擇合適的方法」
- ✅ 區別題：「敏捷與預測的區別」
- ✅ 應用題：「在混合方法中如何應用」

**記憶技巧：**
- 🧠 **預測**：計劃多、變更少、文件多
- 🧠 **敏捷**：迭代多、客戶參與、變更快速響應
- 🧠 **混合**：「一半一半」，根據階段選擇
