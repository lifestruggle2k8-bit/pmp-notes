# 里程碑 vs 任務持續時間 (Milestones vs Task Duration)

里程碑和任務持續時間是時程管理中的兩個重要概念。正確理解它們的區別對於準確規劃和控制專案進度至關重要。

## 📖 原文內容

### 里程碑的定義

**里程碑**是指專案中的重要事件或完成點，通常表示一個階段或重要成果的交付。

### 任務持續時間的定義

**任務持續時間**是指完成一項活動或任務所需的工作時間或日曆時間。

### 里程碑 vs 任務持續時間

| 面向 | 里程碑 | 任務持續時間 |
|------|--------|----------|
| **定義** | 重要的事件或完成點 | 完成活動所需的時間 |
| **持續時間** | 零持續時間（瞬間） | 有具體的時間跨度 |
| **用途** | 標記重要進展 | 規劃和安排活動 |
| **例子** | 需求完成、測試開始 | 開發功能需要10天 |
| **監控焦點** | 達到里程碑的時間 | 任務實際耗時 vs 計劃 |

### 里程碑的特點

- **零持續時間**：里程碑不消耗時間
- **明確的完成條件**：必須清楚定義完成標準
- **易於識別**：應該易於在日程表中識別
- **可衡量**：必須有客觀的衡量標準
- **重要性**：通常代表主要的決策點或交付點

### 持續時間估算

#### 估算方法

| 方法 | 說明 | 適用場景 |
|------|------|--------|
| **專家判斷** | 基於經驗的估算 | 類似專案的依據 |
| **歷史數據** | 基於過去相似任務 | 重複性工作 |
| **類比估算** | 基於類似項目的歷史數據 | 早期規劃階段 |
| **參數估算** | 基於統計關係和歷史數據 | 大型專案 |

#### 三點估算（PERT）

```
樂觀時間（O）+ 可能時間（M）×4 + 悲觀時間（P）
預期時間 = ───────────────────────────────────
                        6

這考慮了不確定性和風險。
```

### 活動序列與里程碑

```
計劃 → 需求分析 → 設計 → 里程碑：設計完成
   ↓          ↓
  5天       10天
     ↓
    開發 → 測試 → 里程碑：上線
     ↓      ↓
    20天   15天
```

## 🎯 重點摘要

- **里程碑 = 關鍵事件**：標記專案中的重要完成點。

- **持續時間 = 工作量**：代表實際需要的時間。

- **都很重要**：里程碑監控進度，持續時間規劃資源。

- **三點估算降低風險**：比單點估算更準確。

- **歷史數據最可靠**：基於實際過去數據的估算最準確。
