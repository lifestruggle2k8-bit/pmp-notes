# 專案經理的角色 (Role of Project Manager)

專案經理是負責領導並協調所有團隊活動以達成專案目標的個人。專案經理的角色遠超出簡單的任務管理，涵蓋領導、溝通、決策和關係管理。

## 📖 原文內容

### 專案經理的定義

**專案經理**是被組織授權以帶領專案團隊實現專案目標的人員。

### 專案經理的核心職責

| 職責 | 說明 |
|------|------|
| **規劃** | 制定專案計畫，定義範圍、時程、預算 |
| **執行** | 帶領團隊執行計畫，完成工作 |
| **監控** | 監督進度，跟蹤績效，進行必要調整 |
| **溝通** | 與各方利害關係人溝通專案信息 |
| **領導** | 激勵團隊，解決衝突，做決定 |
| **管理風險** | 識別、分析和應對風險 |
| **管理變更** | 評估和控制範圍變更 |
| **品質保證** | 確保工作符合品質標準 |

### 專案經理的能力

#### 1. 技術能力（Technical Skills）
- 專案管理知識（PMBOK）
- 行業知識
- 工具和技術使用

#### 2. 管理能力（Management Skills）
- 計劃和組織
- 監控和控制
- 決策制定
- 資源配置

#### 3. 領導能力（Leadership Skills）
- 願景和戰略思維
- 激勵和鼓勵
- 衝突解決
- 決策制定

#### 4. 軟技能（Soft Skills）
- 溝通
- 談判
- 影響力
- 同理心

### 專案經理 vs 功能經理

| 面向 | 專案經理 | 功能經理 |
|------|--------|--------|
| **焦點** | 專案目標 | 部門功能 |
| **時間** | 有開始和結束 | 持續 |
| **權力** | 臨時權力 | 正式權力 |
| **團隊** | 臨時團隊 | 永久團隊 |
| **目標** | 交付專案成果 | 維持運作效率 |

## 🎯 重點摘要

- **專案經理是關鍵角色**：專案成功在很大程度上取決於PM的有效性。

- **多面手能力**：現代PM需要技術、管理和領導能力。

- **軟技能越來越重要**：2026年考試更重視領導、溝通、人員管理。

- **PM是組織的代表**：對利害關係人負責，也代表組織決策。

- **PM需要不斷進化**：適應敏捷、數字轉型等新趨勢。
