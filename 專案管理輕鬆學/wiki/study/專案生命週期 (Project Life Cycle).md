# 專案生命週期 (Project Life Cycle)

專案生命週期是指從專案的啟動到結束的整個過程。理解專案生命週期對於規劃、執行和監控專案至關重要。

## 📖 原文內容

### 生命週期的定義

**專案生命週期**是指從啟動專案到完成專案的一系列階段，每個階段都有特定的活動和交付物。

### 典型的專案生命週期階段

| 階段 | 說明 | 主要活動 |
|------|------|--------|
| **啟動（Initiating）** | 確認專案的需要和可行性 | 商業案例、立項批准 |
| **規劃（Planning）** | 制定實現專案目標的計畫 | 計畫編制、資源規劃 |
| **執行（Executing）** | 執行計畫中的工作 | 生產、測試、交付 |
| **監控（Monitoring）** | 監督進度和績效 | 變更管理、質量控制 |
| **收尾（Closing）** | 正式結束專案 | 交付、評估、知識移交 |

### 不同的生命週期類型

#### 1. 瀑布式（Waterfall）
- 線性、順序進行
- 適合：需求明確、變更少的專案
- 例：基礎設施建設

#### 2. 迭代式（Iterative）
- 多輪重複同樣的步驟
- 適合：需求可逐步細化的專案
- 例：軟體開發

#### 3. 增量式（Incremental）
- 逐步增加功能和特性
- 適合：大型複雜專案
- 例：系統升級

#### 4. 敏捷式（Agile）
- 快速迭代，頻繁交付
- 適合：需求變化、客戶參與度高的專案
- 例：創意設計、應用開發

### 適應性與預測性

```
預測性（Waterfall）
← 變更少，需求明確 ←→ 變更多，需求不確定 →
敏捷性（Agile）
```

---

## 🎯 重點摘要

- **生命週期是框架**：為專案提供結構化的方法。

- **生命週期≠過程組**：過程組是重疊的管理活動，生命週期是順序階段。

- **選擇合適的生命週期很關鍵**：根據專案特性選擇正確的方法。

- **混合方法越來越普遍**：許多組織採用適應預測性方法。
