# Domain I：人員 (People) 33%

Domain I 人員領域著重於專案團隊的建立、領導、管理和發展，強調軟技巧（領導力、溝通、衝突解決）在專案成功中的關鍵角色。

## 📖 原文內容

### Domain I 的核心目標

Domain I 專注於專案人員相關的管理，包括：
- 建立和維持高績效的專案團隊
- 有效的領導和激勵
- 利害關係人的參與和管理
- 溝通和知識轉移
- 衝突解決和協商

### Domain I 的 8 個 Task

| Task 號 | 主題 | 重點 | 重要度 |
|--------|------|------|--------|
| **1** | 建立共同願景 | 確保利害關係人對願景有共識 | ⭐⭐ |
| **2** | 管理衝突 | **永遠選「面對/解決」** | ⭐⭐⭐⭐⭐ |
| **3** | 領導專案團隊 | 情境領導、僕人式領導、授權 | ⭐⭐⭐ |
| **4** | 參與利害關係人 | 識別、分析、參與計畫 | ⭐⭐⭐ |
| **5** | 對齊利害關係人期望 | 分類、識別、促進討論 | ⭐⭐ |
| **6** | 管理利害關係人期望 | 監控滿意度、維持成果 | ⭐⭐ |
| **7** | 確保知識轉移 | 經驗學習、OPA 更新 | ⭐⭐ |
| **8** | 規劃和管理溝通 | 推式/拉式溝通、溝通渠道 | ⭐⭐ |

## 🎯 重點摘要

- **Domain I 佔 33%**：將近三分之一的考題集中在人員管理和領導力上，重點在於軟技巧而非技術技巧。

- **衝突管理是高頻考點**：在 PMP 考試中，衝突題目高達 90% 的答案是「面對/解決（Confrontation/Problem Solving）」，永遠避免「迴避」和「撤退」。

- **8 個 Task 的優先順序**（考試常考）：
  1. Task 2（衝突管理）- 最常考
  2. Task 3（領導）- 常考
  3. Task 4（利害關係人參與）- 常考
  4. Task 8（溝通）- 常考

- **領導風格多樣化**：需要了解情境領導（根據團隊成熟度調整）、僕人式領導（以團隊為中心）、分散型領導（賦權團隊）等不同風格的應用場景。

- **利害關係人管理的關鍵**：使用「權力/關切二維模式」分類利害關係人（控制、顯著、滿意、保持知情），並根據分類調整參與策略。

- **溝通渠道計算公式**：n(n-1)/2，其中 n = 人數。溝通管道越多，衝突風險越高，需要更好的溝通計畫。

- **知識轉移的必要性**：每個專案結束都需要進行經驗學習，更新組織流程資產，為後續專案提供借鑑。

## 📋 考試要點

**考題類型：**
- ✅ 場景題：「團隊成員發生衝突，你應該...」
- ✅ 單選題：衝突解決方式、領導風格選擇
- ✅ 情境題：利害關係人管理、溝通策略

**高頻陷阱：**
- ❌ 衝突題選「迴避」或「強制」 → 應該選「面對/解決」
- ❌ 混淆溝通形式（正式書面、非正式口頭等）
- ❌ 誤解利害關係人分類方法

**記憶技巧：**
- 🧠 **衝突優先順序**：面對 > 妥協 > 強制 > ❌迴避/撤退
- 🧠 **利害關係人矩陣**：Power/Interest 四象限
- 🧠 **溝通渠道**：n(n-1)/2 快速計算
