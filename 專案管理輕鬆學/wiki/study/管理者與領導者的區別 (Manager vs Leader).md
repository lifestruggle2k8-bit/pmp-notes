# 管理者與領導者的區別 (Manager vs Leader)

專案經理需要同時是管理者和領導者。理解兩者的區別和如何結合兩種能力對於有效的專案領導至關重要。

## 📖 原文內容

### 管理者（Manager）vs 領導者（Leader）

| 面向 | 管理者（Manager） | 領導者（Leader） |
|------|-------------------|-----------------|
| **焦點** | 事情和流程 | 人和願景 |
| **方式** | 控制和監督 | 激勵和鼓勵 |
| **目標** | 效率和合規 | 變革和創新 |
| **時間** | 短期、戰術 | 長期、戰略 |
| **權力來源** | 職位和權限 | 信任和尊重 |
| **應對變化** | 管理變化 | 推動變化 |
| **決策** | 基於規則 | 基於願景 |
| **團隊** | 下屬 | 追隨者 |

### 管理者的特點

**活動：**
- 制定計劃
- 組織資源
- 監督績效
- 控制和協調

**技能：**
- 計劃和組織
- 監控和控制
- 問題解決
- 執行能力

**成功標準：**
- 在預算範圍內
- 按時交付
- 符合規範
- 執行効率高

**例子：**
```
管理者說：「我們按計劃進行」
管理者做：確保流程被遵循、資源得到分配、團隊完成任務
```

### 領導者的特點

**活動：**
- 設定願景
- 激勵和鼓勵
- 推動變革
- 設定方向

**技能：**
- 願景思維
- 影響力
- 激勵能力
- 變革領導

**成功標準：**
- 團隊參與度高
- 創新和改進
- 戰略目標達成
- 長期可持續性

**例子：**
```
領導者說：「我們為什麼要做這個，以及它如何改變世界」
領導者做：鼓勵團隊嘗試新想法、面對失敗不氣餒、幫助每個人看到更大的願景
```

### 專案經理需要兩者

```
成功的PM = 管理者 + 領導者

管理能力：
- 制定專案計畫
- 分配資源
- 監控進度
- 控制變更
→ 確保專案按預期執行

領導能力：
- 建立共同願景
- 激勵團隊
- 推動創新
- 帶領變革
→ 確保團隊願意做出卓越成就
```

### 不同情景下的角色轉換

| 情景 | 應該做什麼 | 管理還是領導 |
|------|----------|----------|
| **專案遇到延遲** | 分析原因、制定恢復計劃 | 管理 |
| **團隊士氣低落** | 鼓勵、說明為什麼重要 | 領導 |
| **需求變更** | 評估影響、更新計劃 | 管理 |
| **推動創新方法** | 激勵嘗試、容錯 | 領導 |
| **監控績效** | 追蹤指標、報告進度 | 管理 |
| **應對組織變革** | 帶領團隊適應變化 | 領導 |

### 領導力風格（PMP 2026 強調）

#### 1. 指令式領導（Directing）
- 適用於：新團隊、緊急情況
- 特點：清晰指示、監督

#### 2. 教練式領導（Coaching）
- 適用於：發展團隊
- 特點：指導、反饋

#### 3. 支持式領導（Supporting）
- 適用於：成熟團隊
- 特點：鼓勵、授權

#### 4. 委派式領導（Delegating）
- 適用於：高能力高動力的團隊
- 特點：授權、信任

### 現代 PM 的期望（2026）

```
2026 年，企業期望 PM：
✓ 像經理一樣：計劃、控制、交付
✓ 像領導者一樣：激勵、創新、變革
✓ 像教練一樣：發展、指導、賦權
✓ 像戰略家一樣：思考商業價值、長期影響
```

## 🎯 重點摘要

- **兩者都需要**：既要管理流程，也要領導人。

- **不是非此即彼**：PM 需要靈活地在兩者之間轉換。

- **領導力越來越重要**：2026 考試將領導力提升到核心位置。

- **風格要適應情景**：不同情況需要不同的領導風格。

- **信任是基礎**：領導力基於信任，管理力基於權力。

## 📋 考試要點

**常見題型：**
- ✅ 區別題：「這是管理還是領導活動？」
- ✅ 應用題：「在這個情景中應使用什麼領導風格」
- ✅ 情景題：「PM 應該做什麼」

**記憶技巧：**
- 🧠 **管理**：事情（計劃、控制、監督）
- 🧠 **領導**：人（激勵、願景、變革）
- 🧠 **四種風格**：指令 > 教練 > 支持 > 委派（需要越來越少的監督）
