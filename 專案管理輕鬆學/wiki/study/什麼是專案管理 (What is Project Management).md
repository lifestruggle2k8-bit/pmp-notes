# 什麼是專案管理 (What is Project Management)

專案管理是在專案中應用知識、技能、工具和技術，以滿足專案要求。它涉及將專案活動整合在一起，以確保專案按時、按預算、符合品質標準完成。

## 📖 原文內容

### 專案管理的定義

**專案管理**是指：
> 應用知識、技能、工具和技術於專案活動中，以滿足專案的要求。

### 專案管理的三個核心維度

#### 1. 人員 (People) - 33%
- 領導和管理團隊
- 利害關係人參與
- 衝突管理
- 溝通
- 知識轉移
- 倫理和責任

#### 2. 流程 (Process) - 41%
- 規劃和執行工作
- 管理成本、時程、品質
- 採購和供應商管理
- 監控和控制
- 變更管理

#### 3. 商業環境 (Business Environment) - 26%
- 治理和合規
- 風險管理
- 組織變革
- 外部環境變化
- 持續改善

### 專案管理的五大流程群組

```
啟動流程群組 → 規劃流程群組 → 執行流程群組 → 監控流程群組 → 結束流程群組
(Initiating)  (Planning)    (Executing)    (Monitoring)   (Closing)
```

| 流程群組 | 目的 | 主要活動 |
|---------|------|---------|
| **啟動** | 授權專案、確定目標 | 識別利害關係人、定義願景 |
| **規劃** | 詳細規劃工作 | 制定計劃、定義範疇、估算資源 |
| **執行** | 執行計劃的工作 | 交付成果、管理團隊 |
| **監控** | 追踪進度、控制變更 | 監控成本/時程、品質控制 |
| **結束** | 驗收、歸檔、學習 | 經驗學習、資源釋放 |

### 專案管理的目標

專案管理的終極目標是確保專案：

1. **按時完成** (Schedule)：在預定的期限內交付
2. **按預算完成** (Budget)：不超過批准的成本
3. **符合品質** (Quality)：滿足品質標準和要求
4. **滿足利害關係人** (Stakeholder Satisfaction)：達成期望和需求
5. **創造商業價值** (Business Value)：實現項目目標，為組織帶來價值

### 專案管理的成功標準

**傳統標準（金三角）**：
```
成本 ○
    |\
    | \
    |  \
時程 ○━━━ ○ 範疇
```
- 在預算內
- 按時交付
- 符合範疇

**現代標準（延伸）**：
```
+ 品質達成
+ 利害關係人滿意
+ 團隊發展
+ 商業價值實現
+ 可持續發展
```

## 🎯 重點摘要

- **專案管理是整合藝術和科學**：需要技術知識、人際技巧和業務理解的結合。

- **三維模型（People, Process, Business）**：現代專案管理不只關注時程和成本，還要管理人員、流程和商業環境。

- **五大流程群組貫穿始終**：從啟動到結束，每個階段都很重要，不能跳過。

- **平衡多個目標**：不只是成本/時程/範疇，還要考慮品質、風險、利害關係人滿意度。

- **適應不同開發方式**：預測式、敏捷式、混合式都是有效的專案管理方法，選擇應根據項目特點。

- **持續改進和學習**：每個專案都應進行經驗學習，為組織積累知識和最佳實務。

## 📋 考試要點

**常見題型：**
- ✅ 定義題：「專案管理是什麼？」
- ✅ 流程題：「專案管理的五大流程群組是什麼？」
- ✅ 目標題：「專案成功的標準是什麼？」

**高頻陷阱：**
- ❌ 認為專案管理只關於時程和成本 → 現在還要關注人員和商業環境
- ❌ 忽視某個流程群組 → 五大流程缺一不可
- ❌ 完全按時程表工作就是成功 → 還要符合品質、滿足利害關係人

**記憶技巧：**
- 🧠 **五流程群組**：「啟規執監結」（啟動→規劃→執行→監控→結束）
- 🧠 **三維度**：「人、流程、商業」
- 🧠 **成功標準**：「時程、成本、範疇、品質、利害關係人」
