# 專案治理 (Project Governance)

專案治理是指建立決策框架、權責體系和監督機制，確保專案符合組織戰略和政策，並按預期交付價值。

## 📖 原文內容

### 治理的定義

**專案治理**是指圍繞專案決策、權責分配、監督和績效評估的體系和過程。它確保專案與組織目標對齐。

### 治理的關鍵要素

| 要素 | 說明 |
|------|------|
| **決策框架** | 誰做決定、如何做、什麼時候報告 |
| **權責分配** | 明確的責任和權限 |
| **監督機制** | 定期檢查和監控 |
| **合規性** | 符合政策、法律和標準 |
| **報告體系** | 向利害關係人和高管匯報 |

### 治理結構

```
組織治理（最高）
    ↓
組合治理
    ↓
計劃治理
    ↓
專案治理（執行層）
```

### 治理與管理的區別

| 面向 | 治理 | 管理 |
|------|------|------|
| **焦點** | 方向和決策 | 執行和運作 |
| **層級** | 高管、指導委員會 | 專案經理、團隊 |
| **時間** | 戰略性、長期 | 戰術性、短期 |
| **活動** | 設定標準、監督 | 計畫、執行、監控 |
| **責任** | 確保符合組織戰略 | 確保專案成功 |

### 治理框架的核心職能

1. **立項與授權**：確認專案的商業論證和授權
2. **監督與控制**：定期評審進度和績效
3. **變更控制**：管理範圍、預算、進度變更
4. **績效管理**：評估專案經理和團隊績效
5. **風險與合規**：監管風險和法規遵循

### PMP 2026 新增：治理框架

```
治理層次：
- 公司治理（合規、戰略）
- 組合治理（優先排序）
- 計劃治理（協調）
- 專案治理（交付）
```

## 🎯 重點摘要

- **治理從組織戰略開始**：專案治理必須與組織目標對齐。

- **治理 ≠ 管理**：治理是監督和控制，管理是執行和運作。

- **清晰的決策權**：治理框架應明確誰能做什麼決定。

- **多層級治理**：大型組織有多層級治理體系。

- **2026 重點**：治理與合規、風險、變革密切相關。
