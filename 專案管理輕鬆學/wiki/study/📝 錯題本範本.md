## 🟡 Domain II 錯題（流程）

### EVM（實獲值管理）

#### 題目 3: ________

**原題目**
```
一個專案：
- BAC = $100,000
- PV = $60,000
- EV = $50,000
- AC = $55,000

請計算 SV、CV、SPI、CPI、EAC

A. SV=$10,000, CV=$5,000, SPI=1.2, CPI=0.91, EAC=$110,000
B. SV=-$10,000, CV=-$5,000, SPI=0.83, CPI=0.91, EAC=$110,000 ✅
C. SV=-$10,000, CV=-$5,000, SPI=0.83, CPI=1.09, EAC=$100,000
D. SV=$10,000, CV=$5,000, SPI=1.2, CPI=0.91, EAC=$100,000
```

**計算步驟**

```
✅ 正確計算：

SV = EV - PV = $50,000 - $60,000 = -$10,000 ✅ 落後進度
CV = EV - AC = $50,000 - $55,000 = -$5,000 ✅ 超支成本

SPI = EV / PV = $50,000 / $60,000 = 0.833 ≈ 0.83
CPI = EV / AC = $50,000 / $55,000 = 0.909 ≈ 0.91

EAC = BAC / CPI = $100,000 / 0.91 = $109,890 ≈ $110,000

✅ 答案：B
```

**陷阱分析**
- ❌ 常見錯誤1：搞反 SV 的正負（以為 $60K - $50K = $10K）
- ❌ 常見錯誤2：混淆 SPI 和 CPI（用 AC/EV 計算而非 EV/AC）
- ❌ 常見錯誤3：EAC 計算用錯公式（應該用 BAC/CPI 或 AC + ETC）

**速記表**

| 指標 | 公式 | 含義 | 判斷 |
|------|------|------|------|
| **SV** | EV - PV | 進度差異 | >0 領先 / <0 落後 |
| **CV** | EV - AC | 成本差異 | >0 節省 / <0 超支 |
| **SPI** | EV / PV | 進度指標 | >1 領先 / <1 落後 |
| **CPI** | EV / AC | 成本指標 | >1 節省 / <1 超支 |
| **EAC** | BAC / CPI | 完工預算 | 預估最終成本 |
| **ETC** | EAC - AC | 完工尚需 | 還要花多少錢 |
| **VAC** | BAC - EAC | 完工變異 | 預估節省/超支 |
| **TCPI** | (BAC-EV)/(BAC-AC) | 完工績效指標 | 後續需要達成的 CPI |

**相關概念**
- [[實獲值管理（EVM）]]
- [[成本績效指標 (CPI)]]
- [[時程績效指標 (SPI)]]

### 採購管理

#### 題目 4: ________

**原題目**
```
一個大型基礎設施專案的需求範疇還未完全確定，但客戶急著開始。
專案經理應該選擇哪種合約類型？

A. 固定價格合約（FFP）
B. 成本加固定費用（CPFF）✅
C. 時間與材料（T&M）
D. 固定價格加激勵費用（FPIF）
```

**分析**
- **知識點**：合約類型選擇
- **關鍵詞**：「範疇不確定」+ 「客戶急著開始」
- **正確答案**：B (CPFF)
- **理由**：
  - ❌ A (FFP)：風險太高，範疇不明確不能用固定價格
  - ✅ B (CPFF)：賣方承擔所有風險，適合範疇不確定
  - ⚠️ C (T&M)：用於範疇介於明確和不明確之間
  - ❌ D (FPIF)：需要有清晰的績效目標

**合約類型決策樹**

```
範疇明確？
├─ YES → 固定價格系列（FP, FPIF, FPEPA）
│        └─ 風險由賣方承擔（買方風險低）
│
└─ NO → 成本可償還系列（CPFF, CPIF, CPAF）或 T&M
        ├─ 完全不明確 → CPFF（賣方風險最高）
        ├─ 介於明確和不明確 → T&M
        └─ 不明確但有目標 → CPIF（共享風險）
```

**常見陷阱**
- ⚠️ 看到「範疇不確定」自動排除 FFP
- ⚠️ 看到「客戶急」誤選 T&M（T&M 不一定是答案）
- ⚠️ 混淆 CPFF 和 CPIF

**相關概念**

- [[成本可償還合約]]
- [[時間與材料合約]]

## 🟢 Domain I 錯題（人員）

### 衝突管理

#### 題目 5: ________

**原題目**
```
專案經理發現團隊成員對於新功能的設計意見不合。
一方想要複雜的功能，另一方想要簡單實用的設計。
專案經理應該如何處理這個衝突？

A. 迴避，先不討論，之後再說
B. 強制執行某一方的意見
C. 召集所有相關人員討論，分析雙方需求，找出最佳解決方案 ✅
D. 接受某一方的意見以保持和平
```

**分析**
- **知識點**：衝突管理方式
- **正確答案**：C（面對/解決 - Confrontation/Problem Solving）

**PMP 衝突解決優先順序**

```
1️⃣ 面對/解決（Confrontation/Problem Solving）⭐⭐⭐ 
   └─ 分析根本原因，找出雙贏解決方案
   └─ 考試中 90% 的答案

2️⃣ 妥協（Compromise）⭐⭐
   └─ 雙方都有所讓步

3️⃣ 強制（Force）
   └─ PM 用權力強制執行

4️⃣ 迴避（Avoid）❌ 永遠不選
   └─ 不正面解決問題

5️⃣ 撤退（Withdraw）❌ 永遠不選
   └─ 完全放棄
```

**考試技巧**
> ⭐⭐⭐ **永遠記住**：在 PMP 考試中，衝突題目的答案幾乎總是「面對/解決」！
