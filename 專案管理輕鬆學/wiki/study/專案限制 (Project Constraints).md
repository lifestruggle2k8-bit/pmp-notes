# 專案限制 (Project Constraints)

專案限制，也稱為"鐵三角"或"三重約束"，是指範圍、時程和成本之間的相互制約關係。理解這個概念對於專案規劃和平衡決策至關重要。

## 📖 原文內容

### 鐵三角（Iron Triangle）/ 三重約束

```
        品質
         /\
        /  \
       /    \
      /      \
  時程────────成本
      \      /
       \    /
        \  /
         \/
       範圍
```

### 三個基本限制

#### 1. 時程（Schedule / Time）

**定義：**完成專案所允許的總時間。

**特點：**
- 通常由外部因素決定（市場機會、合約截止日期）
- 難以延長
- 對成本有直接影響

**調整方式：**
- 增加資源（成本上升）
- 減少範圍（質量可能受影響）
- 加班（資源效率下降）

#### 2. 成本（Cost / Budget）

**定義：**完成專案允許的總預算。

**特點：**
- 通常由組織決定
- 受投資回報率限制
- 難以增加

**調整方式：**
- 減少範圍
- 延長時程（但通常不可行）
- 降低質量（風險）

#### 3. 範圍（Scope）

**定義：**專案將交付的產品或服務的邊界。

**特點：**
- 由客戶定義
- 影響時程和成本
- 最容易發生變更

**調整方式：**
- 削減功能
- 分期交付
- 優先排序功能

### 品質的角色（2026 更新）

PMP 2026 將品質提升為與三角等同的地位：

```
        品質
         /\
        /  \
       /    \
      /      \
  時程────────成本
      \      /
       \    /
        \  /
       範圍
```

**品質對其他三者的影響：**
- 品質提升 → 成本上升、時程延長
- 品質降低 → 成本減少、時程縮短（短期）
- 品質是前提條件（不能用品質換取時程）

### 優先權矩陣

| 優先權 | 固定 | 可變 | 可變 | 結果 |
|--------|------|------|------|------|
| 1 | 時程 | 成本 | 範圍 | 時程優先，調整範圍和成本 |
| 2 | 成本 | 時程 | 範圍 | 成本優先，調整時程和範圍 |
| 3 | 範圍 | 時程 | 成本 | 範圍優先，調整時程和成本 |

### 權衡分析

```
案例 1：時程優先（如奧運會）
→ 固定截止日期
→ 可能增加成本
→ 可能削減功能
→ 必須保持品質

案例 2：成本優先（如開發有限預算項目）
→ 固定預算
→ 時程可能延長
→ 可能削減功能
→ 品質受限

案例 3：範圍優先（如定制軟體開發）
→ 必須實現所有功能
→ 時程和成本都可能增加
→ 需要確保品質
```

### 專案經理的責任

1. **識別優先級**：與發起人和利害關係人澄清
2. **傳達權衡**：解釋改變一個會影響其他的
3. **監控平衡**：確保不超出邊界
4. **管理變更**：評估變更對三角的影響

---

## 🎯 重點摘要

- **三角是基本概念**：所有專案都受三重約束。

- **必須優先排序**：無法同時固定所有三個。

- **品質是基礎**：2026 考試將品質提升到同等地位。

- **PM 必須平衡**：在衝突要求之間找到平衡。

- **溝通很關鍵**：讓利害關係人理解權衡。

## 📋 考試要點

**常見題型：**
- ✅ 應用題：「改變時程會影響什麼？」
- ✅ 決策題：「給定優先級，應該調整什麼？」
- ✅ 識別題：「識別限制的優先順序」

**記憶技巧：**
- 🧠 **三角核心**：改一個，動另一個，影響品質
- 🧠 **優先級**：由利害關係人和發起人決定，不是 PM
- 🧠 **2026 更新**：品質是第四維度，不是被犧牲的
