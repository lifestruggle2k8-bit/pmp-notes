# 何謂專案 (What is a Project)

專案是一項臨時的努力，旨在產生獨特的產品、服務或成果。它是組織創造價值、實現戰略目標的重要工具。

## 📖 原文內容

### 專案的正式定義

根據 PMBOK 第七版，**專案**是指：
> 一項**臨時的努力**，為了創造獨特的產品、服務或成果而進行。

### 專案的關鍵特徵

| 特徵 | 說明 | 範例 |
|------|------|------|
| **臨時性** (Temporary) | 有明確的開始和結束日期 | 建造一棟房子，開發一個應用程式 |
| **獨特性** (Unique) | 產出獨特的產品/服務/成果 | 每個建築項目都有不同設計 |
| **有目的的** (Purposeful) | 為了達成特定目標 | 提高市場佔有率、改善流程 |
| **漸進細化** (Progressive Elaboration) | 隨著進展逐步明確細節 | 初期概念模糊，逐漸定義清楚 |
| **有限資源** (Constrained Resources) | 受時間、成本、人員限制 | 預算有限，團隊人數固定 |

### 專案 vs 營運 (Project vs Operation)

| 面向 | 專案 | 營運 |
|------|------|------|
| **定義** | 臨時、獨特的努力 | 持續進行的活動 |
| **時間範圍** | 有明確的開始和結束 | 持續進行，無明確終點 |
| **成果** | 創造新的價值 | 維持既有的價值和運作 |
| **重複性** | 非重複性（每個專案都獨特） | 高度重複性 |
| **變更** | 頻繁，透過變更控制管理 | 透過營運程序管理 |
| **團隊** | 臨時組成，專案結束解散 | 永久組織結構 |
| **目標** | 達成專定目標後結束 | 持續維運 |
| **例子** | 建造新工廠、開發新產品 | 日常製造、客戶服務 |

### 專案的生命週期階段

所有專案通常包含以下四個階段：

```
1. 啟動 → 2. 規劃 → 3. 執行 → 4. 結束
(Initiating) (Planning) (Executing) (Closing)
```

**各階段特點**：
- **啟動**：獲得授權、識別利害關係人、確立願景
- **規劃**：詳細規劃工作、資源、時程、成本
- **執行**：執行計劃的工作、交付成果
- **結束**：驗收成果、歸檔文件、經驗學習

### 專案與策略的關係

```
組織願景 & 策略
      ↓
項目組合 (Portfolio)：所有專案/營運的集合
      ↓
專案 (Project)：實現組織目標的具體努力
      ↓
營運 (Operation)：維持專案成果的日常運作
```

---

## 🎯 重點摘要

- **專案是臨時的**：不是永久性的，有明確的開始和結束日期。一旦達成目標或項目取消，專案就結束。

- **專案是獨特的**：每個專案都會產生獨特的產品、服務或成果，因此需要專門的專案管理方法。

- **專案創造新價值**：不是維持既有狀態，而是為組織帶來新的能力、產品或改進。

- **漸進細化是常態**：專案初期往往不夠明確，細節會隨著進展逐步定義清楚。這不是缺陷，而是專案的特性。

- **專案與營運不同**：必須用不同的管理方式。專案需要靈活應對變化，營運需要穩定和可預測。

- **專案驅動組織戰略**：組織通過執行精心選擇的專案來實現其願景和戰略目標。

- **專案結束後轉入營運**：專案成功交付後，通常由營運團隊接手維持和支持其成果。

## 📋 考試要點

**常見題型：**
- ✅ 定義題：「什麼是專案的特徵？」
- ✅ 區別題：「以下哪個是專案，哪個是營運？」
- ✅ 應用題：「給定情景，判斷是否為專案」

**高頻陷阱：**
- ❌ 認為長期工作就是營運 → 只要有明確結束，就可能是專案
- ❌ 誤解「臨時性」為時間短 → 臨時是指有終點，可以很長
- ❌ 忽視漸進細化 → 項目初期不明確是正常的，不是問題

**記憶技巧：**
- 🧠 **四特徵記憶**：「臨時獨特有目的，漸進資源有限制」
- 🧠 **快速區分**：有終點就是專案，無終點就是營運
- 🧠 **漸進細化**：=「越來越清楚」
