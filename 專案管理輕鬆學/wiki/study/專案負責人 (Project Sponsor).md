# 專案負責人 (Project Sponsor)

專案負責人（發起人）是授權、資助和支持專案的高級管理人員。負責人在專案的成功中扮演關鍵角色，特別是在解決組織層面的問題和支持變革時。

## 📖 原文內容

### 專案負責人的定義

**專案負責人**是指：
- 授權並資助專案的高級管理人員
- 對專案的商業成果負責
- 擁有決定專案繼續或終止的權力

### 負責人的責任

| 責任 | 說明 |
|------|------|
| **商業論證** | 開發和維護商業案例 |
| **資金** | 獲取和管理專案預算 |
| **授權** | 向PM授權和支持決定 |
| **決策** | 做出戰略決策 |
| **障礙清除** | 解決超越PM權限的問題 |
| **支持** | 為團隊提供組織支持 |
| **驗收** | 接受專案的最終交付物 |
| **效益實現** | 確保預期效益實現 |

### 負責人 vs 專案經理

| 面向 | 負責人 | 專案經理 |
|------|--------|--------|
| **權力** | 組織權力 | 專案權力 |
| **時間** | 專案前後 | 專案期間 |
| **焦點** | 商業價值、效益 | 範圍、時程、成本 |
| **責任** | 業務成功 | 專案成功 |
| **參與度** | 定期監察 | 日常參與 |
| **角色** | 所有者 | 經理 |

### 有效發起人的特點

1. **商業敏銳度**：理解商業環境和策略
2. **決策能力**：能夠迅速做出決定
3. **影響力**：在組織內有足夠的影響力
4. **支持**：對PM和團隊的支持
5. **可用性**：定期參與專案
6. **風險意識**：理解和接受風險
7. **變革領導**：推動組織變革

### 發起人面臨的挑戰

```
✗ 時間不足：高管通常忙於其他責任
✗ 決定延遲：影響專案進度
✗ 支持不足：削弱PM的權力
✗ 效益預期：可能過於樂觀
✗ 組織抗拒：應對變革的困難
```

---

## 🎯 重點摘要

- **負責人很關鍵**：專案的成功在很大程度上依賴於有效的發起人。

- **負責人不做PM的工作**：發起人關注商業價值，PM關注執行。

- **早期參與很重要**：發起人應在概念階段就參與。

- **清晰的授權**：發起人應明確授權PM做決定。

- **持續參與**：發起人應定期參與專案，而不是完全離開。

- **效益實現責任**：發起人對實現預期效益最終負責。

**記憶技巧：**
- 🧠 **發起人 = 老闆關係**：授權、資金、支持、決策
