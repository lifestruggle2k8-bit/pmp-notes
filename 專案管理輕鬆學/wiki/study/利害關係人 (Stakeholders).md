# 利害關係人 (Stakeholders)

利害關係人是指對專案有利益關係或受專案影響的個人或組織。專案經理必須識別、分析和管理利害關係人以確保專案成功。

## 📖 原文內容

### 利害關係人的定義

**利害關係人**是指：
- 參與專案或可能受到專案影響的個人或群體
- 可以影響專案或被專案影響

### 常見的利害關係人

| 利害關係人 | 描述 |
|----------|------|
| **發起人** | 授權並資助專案 |
| **客戶/用戶** | 使用或受益於專案成果 |
| **團隊成員** | 執行專案工作 |
| **項目經理** | 領導和管理專案 |
| **組織管理層** | 受專案結果影響 |
| **監管機構** | 確保合規性 |
| **供應商/承包商** | 提供產品或服務 |
| **社區** | 受專案影響的外部方 |

### 利害關係人矩陣（Power/Interest Matrix）

```
高權力，高關注度：
→ 密切管理（MANAGE CLOSELY）
管理策略：保持滿意，定期溝通，尋求支持

高權力，低關注度：
→ 保持滿意（KEEP SATISFIED）
管理策略：定期監控，必要時溝通

低權力，高關注度：
→ 保持知情（KEEP INFORMED）
管理策略：定期匯報進度，回答問題

低權力，低關注度：
→ 監測（MONITOR）
管理策略：最少投入，但保持意識
```

### 利害關係人參與度級別

| 級別 | 說明 |
|------|------|
| **Unaware** | 不知道專案或其影響 |
| **Resistant** | 知道但反對 |
| **Neutral** | 既不支持也不反對 |
| **Supportive** | 支持專案 |
| **Leading** | 主動推動專案 |

---

## 🎯 重點摘要

- **利害關係人識別是首要任務**：不能管理未識別的利害關係人。

- **權力/利益矩陣是實用工具**：幫助優先排序管理方法。

- **參與度很重要**：目標是將所有利害關係人至少提升到「支持」級別。

- **定期溝通是關鍵**：保持利害關係人的參與和支持。

- **利害關係人可能改變**：需要定期重新評估。
