# Agile 專案工具選擇與低科技高接觸哲學

在敏捷（Agile）專案中，工具的選擇直接影響到團隊的溝通頻率與協作氛圍。敏捷管理強烈提倡「低科技、高接觸」（Low-tech, high-touch）優於複雜的電腦模型，在選擇專案工具時，應優先選擇低成本或無成本的工具，並盡量避免需要支付高昂授權費用（Licensing cost）的套裝軟體。

## 一、 實體工具的黃金鐵三角：低科技、高接觸

「低科技、高接觸」是敏捷專案管理的核心主題。

- **推薦使用的實體媒介**：**白板（Whiteboards）、便利貼（Sticky notes）、紙張（Paper）、卡片（Cards）、圖表（Charts）以及辦公室的牆面（Walls）**。
    
- **核心效益**：
    
    - **極低的成本結構**：複雜的專案管理軟體通常需要按人頭支付高昂的「每位使用者授權費（Per user license）」，會大幅增加專案支出。相較之下，實體白板只需幾百美元即可長期使用，便利貼與麥克筆只需幾美元，成本極低。
        
    - **促進溝通與協作**：這些工具能有效實踐敏捷精神，強迫成員離開座位、站在一起進行即時互動。
        

## 二、 使用電腦工具與模型的四大潛在風險

雖然電腦軟體（如 Microsoft Project）功能強大，但在敏捷專案中，盲目依賴電腦模型會帶來嚴重的管理與心理學偏差：

1. **數據準確度的感知偏差 (Perception of Data Accuracy)**：
    
    當使用電腦軟體建立進度表或財務模型時，人們會產生一種**精準的錯錯覺**。誤認為數據既然是透過精密軟體運算並以圖表模型呈現，結果就必然是正確且可靠的。這種對模型的盲目信任，往往會掩蓋實際數據可能存在的巨大誤差。
    
2. **數據品質的「垃圾進，垃圾出」(Garbage In, Gathering Out) 原則**：
    
    無論電腦模型多麼精密，如果期初輸入的數據本身就是錯誤或不完整的，產出的進度表、排程或預算也必然是錯誤的。軟體本身並不保證正確性，它只是處理數據的工具。
    
3. **缺乏團隊互動 (No Interaction)**：
    
    當團隊成員坐在筆電前操作軟體時，使用者的注意力會被完全集中在螢幕與數據輸入上。**這會導致所有人「頭部向下看著螢幕」，而非看向身旁的團隊成員，進而與他人失去實質的交流**。這直接違背了敏捷強調「個人與互動優於流程與工具」的核心原則。
    
4. **「眼不見，心不念」(Out of sight, out of mind)**：
    
    儲存在電腦軟體或雲端系統中的專案資訊，往往會因為隱藏在螢幕和選單後方而被團隊遺忘；只有實體牆面才能提供不間斷的視覺刺激。
    

## 三、 看板與任務板 (Kanban/Task Board) 的實務應用

為了落實低科技高接觸的精神，敏捷強烈建議進行**工具轉換**：

$$\text{【不要使用】電腦上的甘特圖 (Gantt Chart)} \longrightarrow \text{【改用】看板/任務板 (Kanban/Task Board)}$$

```
  【實體牆面/白板/玻璃牆】 ──► 作為「資訊輻射器」 ──► 資訊高效擴散傳播
                                                                │
                                                                ▼
  【每日站立會議核心焦點】 ◄── 抬頭即見 Iteration/Sprint Backlog 視覺化
```

- **作為資訊輻射器 (Information Radiator)**：
    
    實體看板充當了高效的資訊輻射器，其核心功能是確保資訊能夠主動、高效地擴散與傳播到整個團隊。**讓團隊成員即使不主動去查閱或詢問，只要一抬頭，也能隨時察覺、掌握專案的最新現況**。
    
- **多樣化的實作方式**：
    
    團隊可以直接將看板繪製在白板上，甚至直接利用整面牆面空間。形式可以包含：玻璃牆、從天花板到地板的白板牆、或是直接張貼白板壁紙。
    
- **每日站立會議的核心焦點 (Focal Point)**：
    
    看板能將迭代待辦清單（Iteration Backlog）與衝刺待辦清單（Sprint Backlog）變得極其直觀。在進行每日站立會議（Daily Meeting）時，看板提供了一個共同的討論中心與視覺焦點，方便所有人圍繞著真實任務進行高效率的溝通。
    

## 💡 三點需要牢記的部分

1. **「低科技、高接觸」的目的是強迫團隊進行目光交流**：在認證考試與實務工具轉換中，必須深刻理解禁用電腦甘特圖（Gantt chart）的根本原因：**敲擊鍵盤會讓成員「頭向下看」聚焦於螢幕，進而阻斷人際互動**。改用實體看板與便利貼，能強迫成員抬頭、走動、進行面對面的視覺連結與溝通，這才是敏捷文化的靈魂。
    
2. **提防電腦軟體帶來的「錯誤精準感」與「垃圾進出」**：管理者必須時刻保持清醒，切勿盲目信任 Microsoft Project 或複雜財務模型產出的漂亮數據。電腦模型具有掩蓋誤差的毒性（Perception of Data Accuracy），若輸入的原始估算不精準（Garbage In），軟體算出來的死線也必然是錯的（Garbage Out）。實體看板的粗顆粒視覺化，反而能彈性容忍並反映真實的變動性。
    
3. **資訊輻射器的終極標準是「被動接收、一眼看穿」**：高頻考點必須鎖定資訊輻射器（Information Radiator）的特質。雲端軟體容易陷入「眼不見，心不念」的黑箱困境。完美的資訊輻射器必須如同白板牆、玻璃牆一樣裸露在辦公環境中，具備極高的視覺能見度（Visibility），讓利害關係人路過時不需點擊任何滑鼠，就能一眼掌握 Iteration Backlog 的真實消化進度。
