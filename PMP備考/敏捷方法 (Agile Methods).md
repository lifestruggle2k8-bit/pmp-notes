# 熱門敏捷方法論與術語對照指南

## 一、 敏捷是一個傘狀術語 (Umbrella Term)

**Agile（敏捷）** 是一個傘狀術語，它包含了大多數的**增量式 (Incremental)** 或**迭代式 (Iterative)** 開發方法。

- **多樣的方法論**：目前市面上存在超過 12 種以上的敏捷方法論（例如：Lean、Kanban 等）。
    
- **四大核心考點**：在專案管理認證考試中，最常涉及且必須掌握的核心方法論包括：
    
    1. **Scrum**（市場上非常受歡迎，Scrum Alliance 亦提供 CSM 認證）
        
    2. **Extreme Programming (XP)**
        
    3. **Kanban Development**
        
    4. **Lean Software Development**
        

## 二、 最熱門的兩大雙雄：Scrum vs. XP

市場上最受歡迎的兩大敏捷方法論是 **Scrum** 與 **Extreme Programming (XP)**。

### 1. 核心流程的高度相似性

- **[考試重點]**：Scrum 與 XP 的核心開發流程非常相似，**兩者最主要的差異僅在於「術語 (Terminology)」上的不同**。
    
- **應考策略**：在考試中，這兩者的術語有時會被交替或互換使用，在準備與作答時，應將它們視為可以互換的對等概念。
    

### 2. 關鍵術語對照表

|**比較維度**|**Scrum 模式 🌟**|**Extreme Programming (XP) 🛠️**|
|---|---|---|
|**開發週期 (Development Cycle)**|**Sprint (衝刺)**|**Iteration (迭代)**|
|**核心領導角色 (Roles)**|**Scrum Master**|**Coach (教練)**|

## 三、 其他必須掌握的重要敏捷方法論

除了 Scrum 與 XP，以下兩種方法論也廣泛應用於敏捷實務與考題中：

### 1. Lean Software Development (精實軟體開發)

- **核心源起**：源自於**製造業 (Manufacturing)** 的管理概念（如豐田生產方式）。
    
- **核心目標：減少浪費 (Reduce Waste)**。
    
- **實務演進**：透過將製造業消除浪費、優化流程的方法應用於軟體開發中，進而成為敏捷開發的重要基石。
    

### 2. Kanban Development (看板管理)

- **核心特色**：強調將工作流程視覺化，透過控制「在製品數量 (WIP Limit)」來優化團隊的交付效率，同樣是考試與實務中高頻出現的熱門工具。
    

## 💡 三點需要牢記的部分

1. **Scrum 的 Sprint 就是 XP 的 Iteration**：這是考題中最常出現的術語陷阱。不論題目敘述使用的是 Scrum 的 **Sprint** 還是 XP 的 **Iteration**，它們在概念上都代表「1 至 4 週的短衝開發週期」，本質完全一致。
    
2. **Scrum Master 與 Coach 是對等的僕人式領導者**：在角色稱呼上，Scrum 體系定義的 **Scrum Master**，在極限編程 (XP) 中對應的角色即為 **Coach（教練）**。兩者在團隊中皆扮演著排除障礙、服務團隊的教練角色，考題中看到時可直接互換理解。
    
3. **精實開發 (Lean) 的核心關鍵字是「減少浪費」**：只要在情境題中看到「源自製造業」、「消除不必要的流程/文件」、「精簡架構」或「減少浪費 (Reduce Waste)」，其背後的指導思想必定指向 **Lean（精實開發）**。
