### Lead Time 與 Cycle Time 的定義

- **Lead Time**：指某個項目從最開始到整個流程完全結束所需的時間
    - 例如：從需求收集、需求分解、設計、開發、測試到最後部署的完整週期
- **Cycle Time**：通常是 Lead Time 的一個子集
    - 指的是完成整個流程中的某一部分所需的時間

### Cycle Time 的深入理解

- **Cycle Time 的範例**
    - 例如：僅包含「設計與開發」階段所需的時間，這就是 Lead Time 的一部分
- **Cycle Time 與 WIP (Work In Progress) 的關係**
    - Cycle Time 衡量的是「完成工作所需的時間」
    - 它與 **WIP (進行中的工作)** 密切相關
    - **[核心觀念]** Cycle Time 本質上反映了 WIP 的狀態
- **為什麼要極小化 Cycle Time？**
    - 過高的 Cycle Time 通常伴隨著過多的 WIP，這會導致以下問題：
        - 代表資金已投入，但尚未產生投資回報 (No return on investment yet)
        - 會掩蓋流程中的瓶頸 (Hides bottlenecks) 並遮蔽效率問題
        - 會增加風險，可能導致潛在的重工 (Potential rework)

### 過度進行中工作 (Excessive WIP) 的問題

- **[核心關聯]** Cycle Time 衡量的是完成工作的時間，因此它與 WIP (Work In Progress) 密切相關
- 過多的 WIP 會帶來以下問題：
    - **資金回收延遲**：代表資金已經投入，但在目前階段尚未產生任何投資回報 (No return on investment yet)
    - **遮蔽流程問題**：會掩蓋流程中的瓶頸 (Hides bottlenecks)，進而使效率問題變得難以察覺
    - **增加風險**：代表潛在的重工風險 (Potential rework)
        - **[風險的特性]** 風險必須在「開始執行任務」後才會發生，就像你必須先開車上路，才有可能發生車禍一樣

### Cycle Time 的計算公式

- **[核心關聯]** 長 Cycle Time 代表擁有龐大的 WIP (進行中的工作)
- **Cycle Time 計算公式**

    $$\text{Cycle Time} = \frac{\text{WIP}}{\text{Throughput}}$$

- **Throughput (吞吐量)** 的定義
    - 指在給定時間內可以完成的工作量

### Cycle Time 計算實例

- **[計算範例]** 練習如何應用 Cycle Time 公式
    - **題目條件**：
        - 特性 A (Feature A) 的工作量為 60 點 (60 points of work)
        - 團隊的吞吐量 (Throughput) 為每天 5 點 (5 points per day)
    - **計算過程**：
        - 根據公式：$\text{Cycle Time} = \frac{\text{WIP}}{\text{Throughput}}$
        - 代入數值：$\text{Cycle Time} = \frac{60}{5} = 12$
    - **最終結果**：特性 A 的 Cycle Time 為 **12 天**
- **[核心觀念]** 公式背後的邏輯
    - Cycle Time 本質上就是：你要完成的工作量 (WIP) 除以你完成工作的速度 (Throughput)
    - 這種邏輯非常直觀，就像是衡量「剩餘工作量」與「執行速率」之間的關係

### Cycle Time 計算的直覺理解

- **[核心觀念]** Cycle Time 公式其實非常直覺，可以透過日常生活中的例子來理解：
    - **閱讀範例**：
        - 若一份學習指南有 100 頁（WIP = 100 頁）
        - 每分鐘可以讀 1 頁（Throughput = 1 頁/分鐘）
        - 完成閱讀的 Cycle Time = $100 \\div 1 = 100$ 分鐘
    - **駕駛範例**：
        - 目的地距離 120 英里（WIP = 120 英里）
        - 行駛速度為每小時 60 英里（Throughput = 60 英里/小時）
        - 到達目的地所需的時間 = $120 \\div 60 = 2$ 小時
