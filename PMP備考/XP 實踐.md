### Scrum 與 極限編程 (XP) 術語對照

- 為了在不同的敏捷框架間切換，需了解兩者術語的對應關係

| Scrum | Extreme Programming (XP) | 定義 (Definition) |
| --- | --- | --- |
| Sprint | Iteration | 固定長度的時間箱 (Fixed-length period of time / timebox) |
| Release | Small Release | 發佈至生產環境 (Release to production) |
| Sprint/Release Planning | Planning Game | 敏捷規劃會議 (Agile planning meetings) |
| Product Owner | Customer | 專案的業務代表 (Business representative to project) |
| Retrospective | Reflection | 「經驗教訓」風格的會議 ("Lessons learned"-style meeting) |
| ScrumMaster | Coach | 敏捷專案經理 (Agile project manager) |
| Development Team | Team | 獲得授權的跨職能團隊 (Empowered Cross-Functional team) |
| Daily Scrum | Daily Standup | 簡短的每日狀態會議 (Brief daily status meeting) |

- **[重點概念]** Sprint 與 Iteration 是同義詞
    - 指的是一個固定的時間箱 (Timebox)，通常為 1 到 4 週的工作量
