# 管理者 vs. 領導者：核心定義與職能

專案經理在職涯中必須具備「管理」的效率與「領導」的願景，並能視專案性質（如穩定的營建工程或演進中的軟體開發）切換重心。

## 一、 管理 (Management)：確保流程高效執行

管理專注於任務、流程和營運，其核心目標是維持秩序與秩序，並在既定框架內達成預定目標。

- **定義：** 涉及規劃、組織、協調和控制資源。
- **權威來源：** 擁有企業內正式的權威與責任。
- **執行重點：**
    - **規劃與組織：** 決定使用哪些資源（電腦、軟體、人員）並安排工作流程。
    - **協調與控制：** 確保所有工作依計畫進行，並監督流程是否得到遵循。
- **核心目標：** 聚焦於即時目標與營運效率。
- **態度：** 接受現狀，專注於「做對的事」（Do things right），即依循政策精確執行。

---

## 二、 領導 (Leadership)：啟發與影響他人

領導力專注於人的因素，涉及設定方向、激勵團隊，並賦予個人權力朝願景前進。

- **定義：** 專注於啟發、影響與賦權，鼓勵協作與信任。
- **權威來源：** 通常來自人際關係與影響力，領導者可能從部門內自然演進而出。
- **執行重點：**
    - **願景設定：** 擁有長遠願景，不拘泥於日常營運，而是引導團隊朝未來前進。
    - **激勵信任：** 信任員工能完成工作，而非透過強制控制。
    - **質疑與創新：** 詢問「是什麼」與「為什麼」，挑戰不合時宜的現狀。
- **核心目標：** 關注長期目標與人類進步。
- **態度：** 拒絕維持現狀，專注於「做正確的事」（Do the right things），考慮社會與環境福祉。

### 三、管理與領導比較表 (PMI Process Group: A Practice Guide, 第 63 頁, 2023)

| ==管理 (Management)==                       | ==領導 (Leadership)==                                                         |
| ----------------------------------------- | --------------------------------------------------------------------------- |
| 使用職權直接指揮 (Direct using positional power)  | 引導、影響並利用人際關係進行協作 (Guide, influence, and collaborate using relational power) |
| 維持 (Maintain)                             | 發展 (Develop)                                                                |
| 專注於系統和結構 (Focus on systems and structure) | 創新 (Innovate)                                                               |
| 依賴控制 (Rely on control)                    | 專注於人際關係 (Focus on relationships with people)                                |
| 專注於短期目標 (Focus on near-term goals)        | 激勵信任 (Inspire trust)                                                        |
| 問如何以及何時 (Ask how and when)                | 專注於長遠願景 (Focus on long-range vision)                                        |
| 專注於底線 (Focus on bottom line)              | 問是什麼以及為什麼 (Ask what and why)                                                |
| 接受現狀 (Accept status quo)                  | 專注於地平線 (Focus on the horizon)                                               |
| 做對的事情 (Do things right)                   | 挑戰現狀 (Challenge status quo)                                                 |
| 專注於營運問題和問題解決                              | 做正確的事情 (Do the right things)                                                |
|                                           | 專注於願景、協調、激勵和啟發 (Focus on vision, alignment, motivation, and inspiration)    |

## 四、 專案經理的實務應用建議

在現代職場中，PM 必須融合這兩種角色，並根據專案類型靈活切換：

1. **穩定、高風險型專案（如：蓋房子）：**
    - 這類專案不容許隨意挑戰現狀，結構的任何改變都可能導致安全風險。
    - 應以**管理者**角色為主，嚴格控管流程與標準。
2. **適應性、演進型專案（如：軟體開發）：**
    - 範圍經常變化，需要大量激勵與啟發來推動創新。
    - 應發揮**領導者**角色，挑戰既有做事方式，開發新流程。
3. **職涯進階：**
    - 當公司成長時，對領導力的需求會大幅增加。
    - 成為更好的 PM 需要學習大量的人際技能（People Skills），持續激勵團隊並建立信任關係。
