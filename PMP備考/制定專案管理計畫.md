### 專案管理計劃的重要性

- **周密計劃的核心價值**：確保執行工作時無人迷失、無人不知所措，每人清楚了解成功專案所需完成事項
    - 計劃需**晶瑩剔透**（crystal clear）、**超詳細**（super detailed）
    - 盡可能蒐集多人輸入，並經批准
- **課程涵蓋範圍**：透過24個流程來建構此類計劃

### 整合流程（Integration Processes）

- **定義與角色**：主要處理**所有其他流程的組合**，更像是**總結性流程**（summary processes）
    - 在**PMBOK所有版本**中，這些流程是該**流程組所有流程的介紹**
- **範例**：**Develop Project Charter** 屬於整合流程
    - Initiating、Planning、Monitoring & Controlling、Closing 階段各有整合流程
- **Develop Project Management Plan 的本質**：總體層級的**總結性流程**（summary level process）
    - 包含所有其他流程，這些流程是完成此計畫所必需的
    - 透過執行**23個規劃流程**，產生完整的專案管理計畫
- **流程依賴範例**（範疇部分）：
    - **Plan Scope Management** → **Collect Requirements** → **Define Scope** → **Create WBS**
- **其他依賴流程**：
    - **Plan Schedule Management** → **Define Activities** → **Sequence Activities** → **Estimate Activity Durations** → **Develop Schedule**
    - **Plan Cost Management** → **Estimate Costs** → **Determine Budget**
    - **Plan Quality Management**、**Plan Resource Management** → **Estimate Activity Resources**
    - **Plan Communications Management**
    - **Plan Risk Management** → **Identify Risks** → **Perform Qualitative Risk Analysis** → **Perform Quantitative Risk Analysis** → **Plan Risk Responses**
    - **Plan Procurement Management**、**Plan Stakeholder Engagement**

### 各知識領域與流程組的整合模式

- **每個知識領域開頭的整合流程**：先涵蓋一個總結性整合流程，再進入該領域個別子流程
    - **範例對應**：
    - Initiating：**Develop Project Charter**
    - Planning：**Develop Project Management Plan**
    - Executing：**Direct and Manage Project Work**
    - Monitoring & Controlling：**Monitor and Control Project Work**
    - Closing：**Close Project or Phase**
- **相似性觀察**：整合流程名稱反映流程組特性（如planning聽起來像plan、executing關於work、monitoring就是monitor）
    - 所有子流程貢獻於上層整合流程，形成高階總結
- **學習順序**：每個知識領域先介紹整合流程，之後細分必要個別流程

### Develop Project Management Plan 流程細節

- **流程定義**：定義、準備、協調所有計畫組成部分，並將其整合成一個**整合的專案管理計畫**
    - 全面文件，概述**所有專案工作的基礎**及**工作執行方式**
- **文件特性**：可為**摘要**或**詳細**形式
    - 包含**基準**（baselines）和**計畫**（plans）
- **專案管理計畫的複雜性**：不是單一事物，而是**許多許多許多事**（many, many, many things），所有相關元件都屬於此計畫
    - 包含多個管理計畫與基準，如範疇管理計畫、時程基準等

- **文件形式**：即使**小型專案**也可為摘要，但應盡量**詳細**
    - 計畫越詳細，專案成功機率越高
- **包含基準線與計畫**
    - **基準線**：定義工作內容
        - **成本基準線**：預算
        - **時程基準線**：專案所需時間
        - **範疇基準線**：將完成的工作
    - **計畫**：說明如何達成（後續解釋）
- **專案管理計畫主要特性**
    - 概述專案**如何執行、監控與控制，以及結束**
    - **4個基準線**：範疇、時程、成本、**績效衡量**
    - **14個附屬計畫**
    - 由**專案經理、贊助者、功能經理、計畫經理**（或極少數高階主管）**核准**
    - 提供**專案執行指導**
    - **正式書面溝通文件**
    - **僅在產生變更請求並經變更控制委員會核准後變更**
- **強調書面化與批准**：專案管理計畫**不能只存在腦中**，必須**寫下來並獲得批准**
    - 許多人傾向腦中記住，但需正式記錄
- **變更控制原則**：計畫一旦批准，**唯一變更方式**為產生**變更請求**並經**變更控制委員會（Change Control Board）**批准
    - 這是關鍵強調點，避免任意更改
- **管理計畫 vs 基準線區別**
    - **管理計畫**：**如何執行**的文件，提供步驟指南
        - 如**成本管理計畫**：說明如何管理成本
    - **基準線**：定義工作內容本身
        - 如**成本基準線**：預算（已定義的金額）
    - 表格結構：管理計畫在上、基準線在下，強調此差異
- **基準線具體定義**（延續管理計畫 vs 基準線區別）
    - **範疇基準線**：說明**做什麼**（what the project will do）
        - 範例：油漆房間成**亮白漆**（glossy white paint）
        - 或製作**會計應用程式**
    - **成本基準線**：說明**花多少**（how much）
        - 範例：**20萬美元**或**百萬美元**
    - **時程基準線**：說明**花多久**（how long）
        - 範例：**三個月**或**兩年**
- **管理計畫的具體作用**（如何執行指南）
    - **範疇管理計畫**：說明**如何發展範疇**及**如何管理範疇**（確保工作維持在範疇內）
    - **成本管理計畫**：說明**如何發展預算**及**採取哪些步驟確保不超支**
    - 這些管理計畫基本上是**how-to**文件，提供執行步驟
- 後續將透過各流程建立所有這些管理計畫
- **實際上已有管理計畫**：即使未寫下，也存在於腦中
    - 例如知道**收集需求**並**編寫範圍**的過程
    - **執行專案時**也知道該如何進行
- **稍後課堂深入**：目前只需知道存在**管理計畫**
- **管理計畫即操作手冊**：包含具體**如何執行**的步驟
    - **範疇管理計畫範例**：確保專案不脫離範疇的步驟
        - 每日檢查範疇
        - 撰寫**每週工作進度報告**或**每日報告**來追蹤範疇
    - 其他領域如何做：
        - **如何管理資源**
        - **如何確保品質**
        - **如何管理預算**
        - **如何管理風險**
        - **如何管理採購**
- **管理計畫的核心**：**如何管理變更**（如有人提出變更請求時的處理方式）
    - 這些都是**how-to文件**，包含**每一個步驟**
- **考試重點**：**無需死記**專案管理計畫的所有組成部分
    - 只需**理解**它們的本質
    - 看到任何**管理計畫**（考試題或現實），就能**確切知道它是什麼**
- **流程結束過渡**：已涵蓋專案管理計畫，準備介紹**下一個步驟**
- **專案管理計畫外觀**：看似**龐大文件**（**18個組件**、**14個計畫**、**4個基準線**）
    - 實施時確實龐大
- **經驗經理的完美範例**：具**10年經驗**的專案經理
    - 已知**如何建立範疇、時程、成本**
    - 知道**如何從頭到尾管理整個專案**
- **後續說明**：將在各流程中詳細涵蓋這些**18個組件**
- **完美計畫範例**（10年經驗專案經理）
    - 寫出告訴**如何建立範疇**、**如何建立時程**、**如何管理範疇**、**如何管理時程**、**如何管理成本**、**如何管理品質**、**如何管理風險**的計畫
    - 包含**所有程序**、範疇、時程、成本
- **計畫完成後**：**解雇**經驗經理
    - 將計畫交給**高中生**
    - 高中生能**從頭到尾執行專案**，**像有經驗者一樣好**
- **計畫價值**：足夠**獨立**，無需經驗也能執行
    - 證明**詳細計畫**的重要性與完整性
- **高中生執行能力**：憑計畫能**從頭到尾執行、監控、控制並結束專案**，表現**像十年經驗經理一樣好**
    - 計畫包含**每一步驟**與**每個瓶頸**的詳細指導
    - 證明計畫的**獨立性與完整性**，無需經驗也能成功
- **高中生執行時的處理方式**：面對**每一個問題**，**打開計畫書閱讀並遵循**
    - 專案出現**錯誤**或**奇怪情況**時，第一件事就是**拿出專案管理計畫**並**照計畫執行**
- **具體情境範例**
    - **溝通問題**：讀**溝通管理計畫**並遵循
    - **全新風險**出現：讀**風險管理計畫**，計畫會告訴**如何修復**
- **管理計畫的價值**：提供**管理專案的所有步驟**
    - **不能只記在腦中**，必須**存到計畫書中**作為操作指南
- **管理計畫的儲存方式**：**不能記在腦中**，必須**寫入管理計畫文件中**
    - 這樣**任何人**都能**隨時參考**
    - 即使**專案經理不在專案中**，計畫也能提供所有內容
