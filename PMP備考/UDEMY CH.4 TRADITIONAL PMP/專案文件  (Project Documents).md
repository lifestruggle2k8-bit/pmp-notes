## 一、 專案管理計畫 vs. 專案文件

雖然兩者都與專案相關，但其本質完全不同：

- **專案管理計畫 (Project Management Plan)**：描述了專案將**如何**執行、監控與結束的整合性計畫。一旦建立並通過基準（Baseline）審查，未經正式的**變更請求 (Change Request)** 程序不得隨意更改。
    
- **專案文件 (Project Documents)**：用於輔助專案工作的各類紀錄。它們是在執行 49 個流程中建立、使用並**持續更新**的，不屬於專案管理計畫的一部分，因此更新時通常不需要正式的變更請求。
    

---

## 二、 專案管理計畫的 18 個組成部分

根據 PMI 的指南，專案管理計畫包含 10 個子計畫、4 個基準及 4 個描述性組件：

| **類別**                     | **組成部分**                                                                |
| -------------------------- | ----------------------------------------------------------------------- |
| **子計畫 (Management Plans)** | 1. 範疇、2. 需求、3. 時程、4. 成本、5. 品質、6. 資源、7. 溝通、8. 風險、9. 採購管理計畫、10. 利害關係人參與計畫 |
| **管制計畫 (Control Plans)**   | 11. 變更管理計畫、12. 配置管理計畫                                                   |
| **基準 (Baselines)**         | 13. 範疇基準、14. 時程基準、15. 成本基準、16. 績效測量基準                                   |
| **其他描述**                   | 17. 專案生命週期描述、18. 發展方法                                                   |

## 三、 33 項專案文件及其知識領域對應

這些文件隨流程的執行而產生，並作為後續流程的輸入。例如，「活動持續時間估計」是時程管理的輸出，但會變成成本管理中計算預算的輸入。

### 專案文件清單與來源領域：

| **知識領域**                  | **建立之專案文件**                                      |
| ------------------------- | ------------------------------------------------ |
| **整合管理 (Integration)**    | 假設日誌、變更日誌、問題日誌、經驗教訓登記冊                           |
| **範疇管理 (Scope)**          | 專案範圍陳述書、需求文件、需求追溯矩陣                              |
| **時程管理 (Schedule)**       | 活動屬性、活動清單、持續時間估計、里程碑清單、專案日曆、專案時程、時程網路圖、時程數據、時程預測 |
| **成本管理 (Cost)**           | 估算依據、成本估算、成本預測                                   |
| **品質管理 (Quality)**        | 品質控制測量、品質指標、品質報告、測試與評估文件                         |
| **資源管理 (Resource)**       | 實體資源指派、專案團隊指派、資源分解結構 (RBS)、資源日曆、資源需求、團隊章程        |
| **溝通管理 (Communications)** | 專案溝通紀錄                                           |
| **風險管理 (Risk)**           | 風險登記冊、風險報告                                       |
| **利害關係人 (Stakeholder)**   | 利害關係人登記冊                                         |

## 四、 關鍵動態特性與學習建議

1. **ITTO 的循環**：專案文件是連結流程的橋樑。一流程產出的文件往往是另一流程的輸入原料。
    
2. **持續更新**：完成一個特定活動（如測試）時，可能需要同步更新多達 30 份文件。這種高度相互依賴性反映了專案管理的動態本質。
    
3. **包含但不限於**：雖然官方列出 33 項核心文件，但在實際操作中，專案經理可根據需求靈活使用或建立其他輔助文件。
    
4. **考試應對**：在 PMP 等考試中，你需具備辨識「在哪個階段應使用或更新哪份文件」的能力。
    

> **💡 核心觀念：**
> 
> 計畫是「規則」，文件是「證據」。若要改規則，需走變更控制；若要更新證據，則是日常專案工作的正常產出。
