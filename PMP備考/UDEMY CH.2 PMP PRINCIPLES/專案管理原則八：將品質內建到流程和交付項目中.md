## 專案管理原則八：將品質內建到流程和交付項目中

品質不應是事後補救，而應是專案執行過程中的核心組成部分。無論是管理專案還是執行自動化任務，此原則皆一體適用。

### 1. 品質的核心定義

在專案管理中，品質並非抽象的概念，而是有明確的對齊目標：

- **驗收標準的滿足**：品質即是關於滿足可交付成果預先設定的驗收標準。
- **利害關係人的期望**：滿足利害關係者的期望，並實現專案與產品的具體需求。
- **一致性**：確保產出的交付物與利害關係者設定的需求及專案目標高度一致。

### 2. 從流程開始確保品質

公司沒有任何藉口允許賣出品質差的產品。為了達成高品質，必須將品質融入所遵循的流程中：

- **預防性流程**：從流程開始就確保品質，而非依賴事後檢查。
- **適當且有效**：專案品質需確保執行流程是適當且盡可能有效的。
- **全程關注**：在生產交付項目的整個過程中，必須持續關注品質，因為任何部分的失敗都會導致最終品質低落。

### 3. 品質的多維度面向

品質具備多種面向，團隊需全方位考量以下維度：

| 維度 | 說明 |
| 效能 (Performance) | 產品執行其功能的程度（如網站載入速度）。 |
| 符合性 (Conformity) | 是否符合既定的規格與需求標準。 |
| 可靠性 (Reliability) | 產品在特定時間內穩定運作的能力。 |
| 韌性 (Resilience) | 面對壓力或錯誤時的恢復與適應能力。 |
| 滿意度 (Satisfaction) | 終端使用者或利害關係人的主觀感受與評價。 |
| 效率 (Efficiency) | 產出成果與所投入資源的比例是否優化。 |
| 永續性 (Sustainability) | 產品在長期使用中的環境與維護影響。 |

### 4. 驗收標準與指標的應用

為了衡量品質，團隊必須使用具體的指標與驗收標準。

- **驗收條件範例**：
    - 交付物是否完成了特定功能？
    - 外觀是否符合設計要求？
    - 是否以特定的標準方式執行？
- **指標應用（效能範例）**：
    - 若設定網站載入時間需為 1 秒，而實際花費 10 秒，即被視為效能低落的產品。
- **目標導向**：確保交付內容符合客戶與利害關係人的最終目標。若不了解利害關係人的期望，就無法產出好品質的產品。
