### 掙值管理 (EVM) 概述

- **目的**：用來衡量專案目前的執行進度與狀態
- **核心公式**：共有 12 個關鍵公式需要掌握，後續將透過實際情境進行練習

### 完工預算 (Budget at Completion, BAC)

- **定義**：專案最初設定的總預算
- **計算方式**：不需要公式計算，通常是專案起始時既定的金額
- **重要性**：作為後續所有進度與成本評估的基準點

### 計畫價值 (Planned Value, PV)

- **定義**：在特定時間點，根據計畫「應該」完成的工作所具備的金錢價值
    - PV 代表的是計畫中預期應完成的進度量，以金額來衡量
    - 例如：若 PV 為 100，代表目前進度應該已經完成了價值 100 的工作
- **計算公式**：

  $$\text{PV} = \text{計畫完成百分比} \times \text{BAC}$$

- **如何確定計畫完成百分比？**
    - 根據專案的時程規劃來判斷
    - 例如：一個為期 10 天的專案，在第 5 天結束時，計畫完成百分比應為 50%

### 完工價值 (Earned Value, EV)

- **定義**：專案中目前「實際已經完成」的工作所具備的金錢價值
- **計算公式**：

  $$\text{EV} = \text{實際完成百分比} \times \text{BAC}$$

    - 例如：若 EV 為 60 美元，代表你實際上已經完成了價值 60 美元的工作量
- **[核心重要性]**：EV 是整個 EVM 體系中的「關鍵基石」
    - 幾乎所有的後續公式（如成本變異 CV、完工估算 EAC、完工變異 VAC 等）都會用到 EV 作為分子或組成部分
    - **若 EV 計算錯誤，將導致所有後續衍生出的績效指標全部錯誤**

### PV 與 EV 的關鍵差異

| 指標 | 代表意義 | 衡量標準 |
| --- | --- | --- |
| 計畫價值 (PV) | 根據計畫，目前「應該」完成的工作量 | 預期進度 |
| 完工價值 (EV) | 實際上「已經」完成的工作量 | 實際進度 |

### 實際成本 (Actual Cost, AC)

- **定義**：截至目前為止，專案在執行過程中實際已經花費的金錢總額
- **計算方式**：通常沒有特定公式，直接取自實際支出數據

### 成本變異 (Cost Variance, CV)

- **定義**：衡量「實際完成的工作價值 (EV)」與「實際花費的成本 (AC)」之間的差異
- **計算公式**：

  $$\text{CV} = \text{EV} - \text{AC}$$

- **績效判斷標準**：
    - **CV > 0 (正值)**：代表專案目前處於**低於預算 (Under Budget)** 的狀態，是好事
    - **CV < 0 (負值)**：代表專案目前處於**超出預算 (Over Budget)** 的狀態，是壞事

### 成本績效指數 (Cost Performance Index, CPI)

- **定義**：衡量專案每花費一塊錢，能換回多少價值的工作，用來評估成本效率的速率
- **計算公式**：

  $$\text{CPI} = \frac{\text{EV}}{\text{AC}}$$

- **績效判斷標準**：
    - **CPI > 1**：代表效率良好，每一塊錢花出去都能換回超過一塊錢的工作價值
    - **CPI = 1**：代表預算使用完全符合計畫
    - **CPI < 1**：代表效率低下，花費的成本超過了所產出的工作價值

### 時程變異 (Schedule Variance, SV)

- **定義**：衡量「實際完成的工作量 (EV)」與「計畫中應該完成的工作量 (PV)」之間的差異，用來評估時程進度
- **計算公式**：

  $$\text{SV} = \text{EV} - \text{PV}$$

- **績效判斷標準**：
    - **SV > 0 (正值)**：代表專案進度**超前 (Ahead of Schedule)**
    - **SV = 0**：代表專案進度完全符合計畫
    - **SV < 0 (負值)**：代表專案進度**落後 (Behind Schedule)**

### 時程績效指數 (Schedule Performance Index, SPI)

- **定義**：衡量專案達成時程計畫的速率（效率）
- **計算邏輯**：透過 EV 與 PV 的比率，觀察目前進度與計畫進度的比例關係
- **績效判斷標準**：
    - **SPI > 1**：代表進度超前
    - **SPI = 1**：代表進度符合計畫
    - **SPI < 1**：代表進度落後

### 時程績效指數 (Schedule Performance Index, SPI) 的進階理解

- **計算邏輯**：透過 EV 與 PV 的比率，將進度轉化為一個比例（或百分比）
- **績效判斷與範例**：
        - **SPI = 1**：進度完全符合計畫
        - **SPI > 1 (進度超前)**：
                - 例如：SPI 為 1.2，代表專案目前進度**超前了 20%**
        - **SPI < 1 (進度落後)**：
                - 例如：SPI 為 0.8，代表專案目前進度**落後了 20%**

### 完工估算 (Estimate At Completion, EAC)

- **定義**：根據目前的支出速率，對專案結束時的**總成本**進行的「預測」
- **[關鍵考點]**：在考試中，只要題目要求進行**預測 (Forecast)**，通常指的就是計算 EAC
- **計算公式**：

  $$\text{EAC} = \frac{\text{BAC}}{\text{CPI}}$$

    - **邏輯說明**：將原始總預算 (BAC) 除以目前的成本績效指數 (CPI)，以此反映在目前的成本效率下，完成整個專案最終會花多少錢。

### 完工估算 (EAC) 的深度理解與範例

- **預測邏輯**：EAC 反映了根據目前的支出速率，預期在專案結束時會花費的總金額。
- **實例分析**：
        - **低於預算的情況**：
                - 若 $\text{BAC} = 100$，且計算出的 $\text{EAC} = 95$
                - 代表預測最終會比原預算省下 5 元
        - **超出預算的情況**：
                - 若 $\text{BAC} = 1000$，且計算出的 $\text{EAC} = 1100$
                - 代表預測最終會超出預算 100 元

### 完工尚需估算 (Estimate To Complete, ETC)

- **定義**：基於目前的績效表現，預估「完成剩餘工作」還需要投入多少成本
- **[應用場景]**：當老闆問「為了完成這個專案，我們還需要額外準備多少錢？」時，應使用此指標
- **計算公式**：

  $$\text{ETC} = \text{EAC} - \text{AC}$$

    - **邏輯說明**：用「預測的總成本 (EAC)」減去「已經花掉的錢 (AC)」，剩下的就是完成專案還需要的金額

### 完工變異 (Variance At Completion, VAC)

- **定義**：原始總預算 (BAC) 與新的預測總成本 (EAC) 之間的差異，用來衡量專案結束時預計會剩餘多少錢，或是會超支多少錢
- **計算公式**：

  $$\text{VAC} = \text{BAC} - \text{EAC}$$

- **績效判斷標準**：
    - **VAC > 0 (正值)**：代表專案預計會**低於預算 (Under Budget)** 完成，結束時會有剩餘資金
        - 例如：VAC 為 1000 元，代表專案結束時會多出 1000 元的預算
    - **VAC < 0 (負值)**：代表專案預計會**超出預算 (Over Budget)**，需要額外投入資金才能完成
        - 例如：VAC 為 -1000 元，代表專案結束時會超支 1000 元

### 完工績效指數 (To Complete Performance Index, TCPI)

- **定義**：為了在既定的預算內完成剩餘工作，專案必須達到的績效效率
- **核心概念**：衡量「為了完成剩下的工作，我們接下來必須跑多快」

### 完工績效指數 (To Complete Performance Index, TCPI) 的深度解析

- **定義**：為了在既定的預算 (BAC) 內完成專案剩餘的工作，專案團隊接下來必須達到的績效效率
- **核心邏輯**：這是一個衡量「為了完成剩下的工作，我們接下來必須跑多快」的指標，反映了達成預算目標所需的「努力程度」
- **計算公式**：

  $$\text{TCPI} = \frac{\text{BAC} - \text{EV}}{\text{BAC} - \text{AC}}$$

    - **分子 (BAC - EV)**：代表專案剩餘的工作價值（還剩下多少錢的工作沒做）
    - **分母 (BAC - AC)**：代表剩餘可用的預算（還剩下多少錢可以花）
    - **公式含義**：計算的是「剩餘工作價值」與「剩餘可用預算」的比率
- **績效判斷與實務意義**：
        - **TCPI = 1**：代表必須維持目前的支出效率，才能在預算內完成專案
        - **TCPI < 1 (例如 0.8)**：
                - 代表情況良好。專案只需要以目前 80% 的強度（或效率）工作，就能在預算內完工
        - **TCPI > 1 (例如 1.2)**：
                - 代表挑戰增加。專案必須比目前的效率提升 20%（即必須「更努力」地工作），才能在預算內完成專案
