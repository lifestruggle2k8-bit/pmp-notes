# 規劃撲克 (Planning Poker) 終極實戰指南

在敏捷估算實務中，**規劃撲克（Planning Poker）** 是一種在 Agile 專案（特別是 Scrum 框架）中極其常見且熱門的遊戲化技術，專門用於為產品待辦清單（Product Backlog）中的使用者故事（User Stories）指派相對點數。

## 一、 規劃撲克的三大核心特性

規劃撲克之所以能成為敏捷團隊的首選估算工具，源於它完美融合了以下三項管理學特性：

- **🕵️ 匿名性 (Anonymity)**：整個出牌與思考過程傾向於嚴格的雙盲匿名進行，**與寬頻德爾菲（Wideband Delphi）方法的底層防禦機制非常相似**。
    
- **🤝 協作與高參與度**：具備高度互動與協作性質（High touch），能有效帶動全體成員百分之百投入討論，是激發團隊參與度（Engagement）的絕佳方式。
    
- **🛠️ 低技術門檻 (Low Tech, High Touch)**：不需要任何複雜、高昂的數位軟體或硬體設備，僅需一套紙牌即可在任何實體空間中流暢運作。
    

## 二、 規劃撲克的標準執行流程

規劃撲克的生命週期是一個高度節奏化、防禦帕金森定律與群體盲思的動態循環：

```
 🃏 Planning Poker 標準估算工作流：
 
  【1. 準備發牌】 ──► 每位專家獲得一組印有費波那契數列（1, 2, 3, 5, 8, 13, 21）的實體卡片。
         │
         ▼
  【2. 讀題說明】 ──► 主持人（Facilitator / PO）大聲讀出一則 User Story。
         │
         ▼
  【3. 沉默思考】 ──► 團隊保持沉默思考（30秒至數分鐘，取決於團隊動態），在心中決定點數。
         │
         ▼
  【4. 盲蓋暗牌】 ──► 成員各自選出一張卡片【蓋在桌上，不翻轉數字】，確保完全的匿名性。
         │
         ▼
  【5. 同步揭曉】 ──► 主持人倒數計時（5, 4...0），所有人【同時翻開卡片】，公開估算值。
```

> 🗣️ **工作坊臨床讀題範例**： Product Owner：「各位，我們來看這則故事──_作為一名應收帳款人員（Accounts Receivable Personnel），我希望能輸入發票（Invoice），以便我能準時收到款項，或讓客戶知道他們欠我們多少錢。_」

## 三、 揭曉後的決策科學：如何收斂並達成共識？

當所有人同時翻開卡片後，主持人與團隊必須依據牌面的分佈情況，採取不同的收斂與決策邏輯：

### ⚖️ 狀況 A：存在明確的多數決 (Majority 共識)

當牌面呈現出明顯的群體傾向時，敏捷允許直接採用多數決進行快速收斂。

- _【臨床情境範例】_：團隊共有 8 位跨職能成員。
    
    - **5 位** 成員選擇了 **5 點**
        
    - 2 位成員選擇了 3 點
        
    - 1 位成員選擇了 8 點
        
- _【決策方式】_：由於**多數成員（Majority）都高度收斂在 5 點**，因此該項 User Story 的最終估算值會直接被記錄為 **5 點**，隨後直接進入下一題。
    

### 🛡️ 狀況 B：缺乏明確共識 (點數分散) ──► 採取「最保守防禦」

當點數極度分散、沒有任何一個數字取得壓倒性的多數優勢時，團隊必須啟動風險防禦機制。

- _【臨床情境範例】_：團隊成員對於第二個 User Story 的估算結果如下：
    
    - **3 位** 成員選擇了 **5 點**
        
    - **3 位** 成員選擇了 **8 點**
        
    - 2 位成員選擇了 3 點
        
- _【決策方式】_：點數在 3、5、8 之間拉鋸，無明確多數。此時應採取最保守、最安全的做法──**直接採用該組數字中的最高值：8 點**！
    
- _【管理目的】_：**確保在估算早期為團隊預留足夠的風險緩衝空間（Buffer），鐵腕避免低估未知的開發工作量。**
    

## 💡 三點需要牢記的部分

1. **暗牌蓋桌、倒數同翻，規劃撲克的匿名性是擊碎 HIPPO 的低成本盾牌**：在備考情境題與實務控場時，必須死磕「蓋牌不翻轉」與「倒數同步翻牌」這兩個關鍵動作。這套遊戲機制本質上就是 **Wideband Delphi** 技術的延伸實踐。它透過簡單的實體動作隔絕了視線干擾，確保成員在出牌前無法偷看資深老手或威權主管的點數，徹底消滅了從眾效應（Bandwagon effect）與 **HIPPO（薪水最高者意見）** 對估算主權的侵害。
    
2. **多數決看 Majority，分散時「從高從寬」是保護自組織產能的剛性邊界**：請精確鎖定規劃撲克在面對分歧時的兩大數學決策邏輯。如果牌面有明確的 Majority，可直接記錄多數值；若點數呈現僵局（如 3位出5、3位出8），**最標準的敏捷防禦動作是直接選擇最高點數（8點）**。這完美呼應了理想時間（Ideal Time）與敏捷規劃原則──估算必須具備包容性，主動為潛在風險與干擾預留緩衝，拒絕讓團隊在期初因為過度樂觀而滑向紅區焦慮。
    
3. **Low-tech, high-touch 的實體卡片，是催化「隱性知識」滲透的最高工法**：結合先前學過的**團隊空間（Team Spaces）**環境學，規劃撲克正是 **Low-tech, high-touch（低技術、高互動）** 工具的典範。它不依賴死板的線上表單，而是強迫成員聚集在 10 公尺的物理距離內。當倒數結束、卡片翻開，高低點兩端的強烈對比會立刻啟動**建設性分歧（Constructive Disagreement）**，逼迫專家展開對話（Conversation），進而讓未被記錄的隱性知識（Tacit Knowledge）在圓桌上快速流動。
